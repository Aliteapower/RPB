## 3. Route

- `/stores/:storeId/walk-ins/direct-seating`

The root route redirects to the same page using `VITE_DEFAULT_STORE_ID` when present, or a placeholder UUID when not configured.

## 4. API Client

Created `src/api/walkInDirectSeatingApi.ts`.

- Endpoint: `POST /api/v1/stores/{storeId}/walk-ins/direct-seating`
- Header: `Idempotency-Key`
- Handles success response
- Handles API error envelope
- Handles network failure
- Preserves idempotency replay data from the backend response
- Does not send `tenantId` in the request body

Types align with the WalkIn Direct Seating API contract.

## 6. WalkIn Direct Seating Page

Created `src/pages/WalkInDirectSeatingPage.vue`.

The page supports:

- `partySize`
- `customerName`
- `customerNickname`
- `phoneE164`
- `tableId`
- `tableGroupId`
- `overrideReasonCode`
- `overrideNote`
- Generated `Idempotency-Key`
- API submit
- Success result display
- Error `code` and `messageKey` display

No Reservation, Queue, Cleaning, Turnover, Customer search, Table management, POS, Payment, Marketing, or Membership UI was created.

## 7. Mobile-first Handling

- Single-column layout
- `partySize` is the primary visible field
- Customer, Resource, and Override fields are collapsed by default
- Submit button is large and fixed within the form flow
- Success result displays resource, `seatingId`, status, and idempotency replay flag
- Error result displays stable error `code` and i18n `messageKey`
- No complex table map
- No drag-and-drop table layout

## 8. Idempotency-Key Handling

- The page generates a new idempotency key for each submit.
- The API client sends the key in the `Idempotency-Key` header.
- The request body does not contain `tenantId`.
- The response renders backend idempotency status and `replayed=true` when returned.

## 9. Form Validation

Frontend validation covers:

- `partySize > 0`
- `phoneE164` must match E.164 format when present
- `tableId` and `tableGroupId` cannot both be present
- Submit generates an `Idempotency-Key`

Backend validation remains the source of truth.

## 10. Error Display

The page displays:

- `error.code`
- `error.messageKey`

It does not replace backend `messageKey` with hardcoded business display copy.
