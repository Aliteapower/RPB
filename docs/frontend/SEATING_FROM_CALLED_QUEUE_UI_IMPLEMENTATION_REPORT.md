## 4. API Client

- Endpoint: `POST /api/v1/stores/{storeId}/queue-tickets/{queueTicketId}/seating/direct`
- Idempotency-Key: generated by the page and sent as `Idempotency-Key`
- Body allowlist: `tableId`, `tableGroupId`, `overrideReasonCode`, `overrideNote`, `note`
- Blank optional fields are omitted by the API client.
- `storeId` and `queueTicketId` are used only in the path, not in the body.
- Errors preserve `error.code` and `error.messageKey`; unknown or invalid responses fall back to `UNKNOWN_ERROR / queue.seat.unknown_error`.

## 7. Resource Selection Validation

- `tableId` and `tableGroupId` are strict XOR.
- Both empty returns local `RESOURCE_SELECTION_REQUIRED / queue.seat.resource_selection_required`.
- Both filled returns local `RESOURCE_SELECTION_CONFLICT / queue.seat.resource_selection_conflict`.
- Submit is disabled until `queueTicketId` is present and exactly one resource is selected.
- Backend remains the final validator for resource existence, scope, capacity, locks, and TableGroup validity.

- `alreadySeated=true` renders inside the success panel.
- It is not displayed as a failure.
- Empty `events` display as `[]`.
- Existing `seatingId`, `resourceType`, `resourceId`, queue number, and idempotency fields remain visible.

## 12. Error Display

- Error panel displays `错误代码：{{ apiError.error.code }}`.
- Error panel displays `消息键：{{ apiError.error.messageKey }}`.
- The UI does not replace backend `messageKey` with hardcoded business copy.
- Network or invalid response fallback is `UNKNOWN_ERROR / queue.seat.unknown_error`.

## 13. Mobile-first Handling

- Single-column shell.
- `排队票 ID` field is first and visually prominent.
- `桌台 ID / 桌组 ID` resource selection is immediately below the ticket id.
- Optional override and note fields are inside secondary details blocks.
- Submit button is thumb-friendly and disabled during loading or invalid form state.
- Success result emphasizes `排队号码`, `seated` statuses, `seatingId`, and resource.
- Error result wraps long message keys safely.
- No complex queue workbench, display screen, table map, drag-and-drop, or desktop-only layout was introduced.
