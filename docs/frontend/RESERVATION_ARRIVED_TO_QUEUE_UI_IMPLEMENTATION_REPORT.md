### 4. API Client
- Endpoint: `POST /api/v1/stores/{storeId}/reservations/{reservationId}/queue`
- Idempotency-Key: generated by the page and sent as `Idempotency-Key`
- Body allowlist: `partySizeGroup`, `reasonCode`, `note`
- Blank optional fields are omitted by the API client.
- `storeId` and `reservationId` are used only in the path, not in the body.
- Errors preserve `error.code` and `error.messageKey`; unknown or invalid responses fall back to `UNKNOWN_ERROR / reservation.queue.unknown_error`.

### 6. Form Fields
- Required: `预约 ID` (`reservationId`)
- Optional: `人数分组` (`partySizeGroup`), `原因代码` (`reasonCode`), `备注` (`note`)
- Forbidden: no tenant, store body, reservation body, table, seating, walk-in, cleaning, turnover, no-show, cancellation, queue-ticket-number, or status input is rendered.
- Submit is disabled when `reservationId` is blank or a request is in flight.

### 7. partySizeGroup Behavior
- Options: `自动推导`, `1-2`, `3-4`, `5-6`, `7+`
- Default: `自动推导`
- `自动推导` sends no `partySizeGroup` body field because the page maps it to `null` and the API client omits blank optional fields.
- Explicit values send the selected group code.
