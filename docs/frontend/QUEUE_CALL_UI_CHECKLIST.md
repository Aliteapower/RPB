## 5. API Call Contract

- [x] Future UI calls `POST /api/v1/stores/{storeId}/queue-tickets/{queueTicketId}/call`.
- [x] `storeId` comes from the route path.
- [x] `queueTicketId` comes from required form input and is used only in the path.
- [x] `Idempotency-Key` is generated by the UI per submit.
- [x] Request body allows `calledAt`.
- [x] Request body allows `reasonCode`.
- [x] Request body allows `note`.
- [x] Blank optional fields must be omitted or normalized consistently.
- [x] Request body must not include `tenantId`.
- [x] Request body must not include `storeId`.
- [x] Request body must not include `queueTicketId`.
- [x] Request body must not include `reservationStatus`.
- [x] Request body must not include `tableId`.
- [x] Request body must not include `tableGroupId`.
- [x] Request body must not include `seatingId`.
- [x] Request body must not include `cleaningId`.
- [x] Request body must not include `turnoverId`.
- [x] Request body must not include `skipReason`.
- [x] Request body must not include `rejoinReason`.
- [x] Request body must not include `status`.

## 6. Form Contract

- [x] Optional field: `calledAt`.
- [x] Optional field: `reasonCode`.
- [x] Optional field: `note`.

- [x] `queueTicketId` must be the most prominent input.
