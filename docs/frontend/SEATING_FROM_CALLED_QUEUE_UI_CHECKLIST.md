## 3. Route Contract

- [x] Future route is `/stores/:storeId/queue-tickets/seating/direct`.
- [x] Future page is `SeatingFromCalledQueuePage.vue`.
- [x] Route remains manual QueueTicket seating, not a queue workbench.
- [x] Route does not imply Queue list.
- [x] Route does not imply Queue Display.
- [x] Route does not imply Queue Skip.
- [x] Route does not imply Queue Rejoin.
- [x] Route does not imply Table map.

## 4. Staff Home Contract

- [x] Future entry label is `排队入座`.
- [x] Future entry target is `/stores/:storeId/queue-tickets/seating/direct`.
- [x] Entry appears only for visible/enabled `reservation_queue`.
- [x] Entry appears only when permissions contain `queue.seat`.
- [x] Entry is hidden when `reservation_queue` is missing.
- [x] Entry is hidden when `reservation_queue.entryVisible` is false.
- [x] Entry is hidden when `queue.seat` is missing.
- [x] Entry does not add Queue list, Queue Display, Queue Skip, Queue Rejoin, Table map, Auto assignment, No-show, Cancellation, Cleaning, or Turnover navigation.

## 5. Permission Contract

- [x] UI permission check is display-only.
- [x] Backend App Gate remains final authorization.
- [x] Backend gate remains `app_key = reservation_queue`.
- [x] Backend gate remains `permission = queue.seat`.
- [x] UI does not treat hidden buttons as security enforcement.

## 6. API Call Contract

- [x] Future call is `POST /api/v1/stores/{storeId}/queue-tickets/{queueTicketId}/seating/direct`.
- [x] `storeId` comes from route path.
- [x] `queueTicketId` comes from required form input and is used in path.
- [x] `Idempotency-Key` is generated by UI.
- [x] Suggested idempotency key prefix is `queue:seat:<uuid>`.
- [x] Request body allowlist is `tableId`, `tableGroupId`, `overrideReasonCode`, `overrideNote`, `note`.
- [x] Blank optional fields should be omitted or normalized consistently.
- [x] Request body must not include tenant, store, actor, state, reservation, seating, cleaning, turnover, no-show, cancellation, skip, or rejoin fields.

## 7. Form Contract

- [x] Required field: `queueTicketId`.
- [x] Resource fields: `tableId` and `tableGroupId`.
- [x] Optional fields: `overrideReasonCode`, `overrideNote`, `note`.
- [x] Generated field: `Idempotency-Key`.
- [x] `queueTicketId` is visually primary.
- [x] Resource selection appears before optional notes.
- [x] Submit is disabled while loading.
- [x] Submit is disabled when `queueTicketId` is blank.
- [x] Submit is disabled when resource selection is invalid.
- [x] New submit uses a new idempotency key unless intentionally retrying the same command.

## 8. Resource Selection Contract

- [x] Exactly one of `tableId` or `tableGroupId` is required.
- [x] `tableId` only is valid.
- [x] `tableGroupId` only is valid.
- [x] Neither resource is invalid.
- [x] Both resources are invalid.
- [x] Missing selection may show `RESOURCE_SELECTION_REQUIRED`.
- [x] Conflicting selection may show `RESOURCE_SELECTION_CONFLICT`.
- [x] Backend remains final for capacity, availability, lock, TableGroup validity, tenant scope, and store scope.
- [x] UI does not auto-select a resource.
- [x] UI does not recommend a resource.
- [x] UI does not render a table map.
